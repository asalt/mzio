# mzio

Standalone home for the mzML and MS/MS tooling extracted from `universal-tui`.

## Current commands

- `plot` — export an `mzML` or `ms2` spectrum as SVG, with optional Rust-side fragment annotation for a supplied peptide
- `plot-survey` — export run-level TIC/BPC, MS1 map, and opt-in DDA QC overview figures as clean SVG/PNG files
- `diagnostics` / `diag` — scan centroided MS2 spectra for diagnostic ions and pairwise mass shifts, then write a TSV summary
- `spectra` — browse an `mzML` file in a local TUI with on-disk index caching, chromatogram/map panes, and quick SVG/PNG export

## Run

From this directory:

```bash
cargo run -- --help
cargo run -- plot --help
cargo run -- plot-survey --help
cargo run -- diagnostics --help
cargo run -- spectra --help
```

## Local demo assets

- `assets/demo.mzML`
- `assets/demo.ms2`

Example:

```bash
cargo run -- plot \
  --mzml assets/demo.mzML \
  --scan 2 \
  --peptide "PEPTIDEK/2" \
  --svg /tmp/demo-annotated.svg
```

`ms2` example:

```bash
cargo run -- plot \
  --ms2 assets/demo.ms2 \
  --svg /tmp/demo-ms2.svg
```

Spectra browser example:

```bash
cargo run -- spectra \
  --mzml assets/demo.mzML
```

Run-level survey export example:

```bash
cargo run -- plot-survey assets/demo.mzML
```

DDA survey export example:

```bash
cargo run -- plot-survey assets/demo.mzML \
  --view dda \
  --png
```

Rebuild the browser cache explicitly:

```bash
cargo run -- spectra \
  --mzml assets/demo.mzML \
  --reindex
```

## Plot notes

- v1 annotation is Rust-only and does not require Python.
- Sequence scope is intentionally narrow for now: amino-acid letters with optional inline mass shifts like `M[+15.9949]`, `S[+79.9663]`, or leading N-term shifts like `[+42.0106]PEPTIDE`, plus an optional `/charge` suffix.
- Fixed mods are not assumed automatically; use repeatable `--mod <position>:<delta>` when needed.
- If an inline mass shift omits the sign, it is treated as positive.
- Precursor checking now allows isotope-error windows via `--isotope-errors`, with `0,1,2` as the default.
- `--neutral-losses` enables common residue-aware `-H2O` and `-NH3` fragment variants, plus phospho `-H3PO4` on phospho-Ser/Thr-containing fragments.
- Neutral-loss labels are display-filtered by default at `3%` of base peak; tune this with `--neutral-loss-min-frac`.
- Annotated plots include a full right-hand theoretical ion table with matched ions colored and missing ions retained for comparison, plus a denser ruler-style x-axis.
- Prefer `--scan` for instrument scan numbers. `--index` is the zero-based spectrum position, and `--id` is for broader native-id matching.
- If you omit `--svg`, the autogenerated filename includes the source stem, scan number, peptide+charge when present, and whether neutral losses were enabled. Use `--svg-prefix calibrated` for an extra label.

## Diagnostics notes

- This is exploratory evidence gathering, not PTM localization or a full search engine.
- v1 scans only centroided MS2 spectra and skips profile spectra.
- Peak filtering is intentionally simple: first apply `--min-rel`, then keep the top `--top-n` peaks by intensity.
- Built-in panels are small and interpretable on purpose: `oxonium`/`glyco` add common glyco-associated target ions, and `phospho` adds the `97.976896 Da` phosphate neutral-loss-like delta.
- Output is a flat TSV so it can be grepped, sorted, joined, or plotted downstream without needing a heavier result schema.

## Survey plot notes

- `plot-survey` is non-interactive and writes run-level overview files into `exports/` by default, instead of exporting the currently selected spectrum from the browser.
- The default view writes separate TIC/BPC and MS1 base-peak map SVGs. Use `--out-dir` and `--prefix` to control file locations and stems.
- `--view dda` writes separate DDA acquisition-density and precursor-survey SVGs. Use `--view all` to write every implemented survey plot.
- The map is metadata-based in this first pass: each MS1 scan contributes its base-peak m/z and base-peak intensity.
- DDA plots are also metadata-based: MS2 density, cycle summaries, selected-ion m/z, charge, injection time, and selected-ion intensity are parsed from mzML tags without decoding peak arrays.

## Spectra browser notes

- The browser uses a local `.mzio.idx.gz` cache next to the input file by default.
- Override that location with `--mzml-cache-path`, or disable cache writes with `--no-mzml-cache`.
- The right pane cycles through spectrum, chromatogram, base-peak map, and file-overview views.
- SVG/PNG export stays local and does not require any external service.
